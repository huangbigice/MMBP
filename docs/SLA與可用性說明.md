# 服務等級協議（SLA）與可用性說明

**文件編號**：SLA-001  
**版本**：1.0  
**生效日期**：________  
**適用產品**：智慧股票投資決策支援系統（MMBP 後端 API + FMFA 前端平台）

---

## 一、文件目的

本文件旨在為採購本系統之機構客戶，提供服務等級（Service Level）與可用性目標之明確說明，作為合約設計與責任劃分之依據。若本系統對外收費或採訂閱制營運，本 SLA 將納入服務條款或獨立 SLA 附約。

---

## 二、服務範圍定義

本 SLA 涵蓋之服務包含：

| 服務項目 | 說明 |
|---------|------|
| **MMBP 後端 API 服務** | 股票資料抓取、技術指標計算、預測模型推論、回測引擎、LLM 投資助理對話（SSE 串流）等 REST/Streaming API |
| **FMFA 前端平台** | 股票分析介面、K 線／KD／MACD／成交量圖表、搜尋與投資建議展示 |
| **相依外部服務** | 金融資料來源（如 Yahoo Finance）、LLM 服務（Ollama），其可用性不列入本 SLA 保證範圍 |

---

## 三、目標服務指標（現行目標值）

> **說明**：以下為本公司努力達成之**目標**，非具法律拘束力之承諾。實際承諾等級以正式簽署之 SLA 附約為準。

### 3.1 可用性目標

| 指標 | 目標值 | 計算方式 |
|------|--------|----------|
| **每月可用性** | 99% | (總分鐘數 − 非計畫中斷分鐘數) ÷ 總分鐘數 × 100% |
|planned 維護時間 | 不計入中斷 | 於計畫維護窗口內進行之維護不計入不可用時間 |

### 3.2 回應時間目標

| API 類型 | P95 回應時間目標 | 備註 |
|----------|------------------|------|
| 健康檢查 `GET /api/v1/health` | < 200 ms | 用於監控告警 |
| 股價／指標查詢 | < 3 秒 | 含外部資料抓取 |
| 預測 API `POST /api/v1/predict` | < 5 秒 | 含模型推論與評分 |
| 回測 API | < 30 秒 | 依區間與標的數量而異 |
| LLM 串流對話 | 首 byte < 3 秒 | SSE 串流，持續輸出直至完成 |

### 3.3 計畫維護窗口

| 項目 | 說明 |
|------|------|
| **維護時段** | 每月第 2、4 週之週日凌晨 02:00–06:00（台灣時間 UTC+8） |
| **通知方式** | 至少 72 小時前以 Email 或官方公告通知 |
| **緊急維護** | 若遇安全性或重大異常，得於較短通知期內執行，並於事後補發說明 |

## 四、通知與溝通機制

| 情境 | 通知方式 | 時效 |
|------|----------|------|
| 計畫維護公告 | Email、官網公告、API 狀態頁（若有） | 72 小時前 |
| 非計畫中斷 | Email、簡訊（若客戶已提供）、官網公告 | 偵測到後 30 分鐘內 |
| 恢復正常 | 同上 | 恢復後 15 分鐘內 |
| 重大事件事後報告 | Email 或書面報告 | 事件結束後 5 個工作天內 |

## 五、責任上限與免責事項

### 5.1 責任上限

- 除因故意或重大過失所致之損害外，本公司對本服務所生之損害賠償責任，以**客戶於該事件發生前 12 個月內所支付之服務費用總額**為上限。
- 間接損害（含營業損失、利潤損失、資料遺失所生之衍生損失）不予賠償，法律另有規定者除外。

### 5.2 免責範圍

以下情況不計入 SLA 違約或不可用時間：

- 客戶端網路、設備或操作不當所致之中斷
- 第三方服務（金融資料來源、LLM 服務等）之不可用或延遲
- 不可抗力（天災、戰爭、政府行為、大規模網路攻擊等）
- 客戶未依合理使用政策使用服務，或超出約定之請求量或規格

## 六、過渡條款（若尚未能保證 SLA）

若本系統尚未具備足以承諾上述 SLA 之基礎建設，得以以下條款代之：

> **本公司目前不提供具法律拘束力之 SLA 承諾。服務將以「盡商業上合理努力」（Commercially Reasonable Efforts）方式提供。本公司會持續優化基礎建設與監控，於適當時機推出正式 SLA 並以書面通知客戶。**

此過渡條款應載明於服務條款或獨立說明頁，讓客戶了解現況，並便於日後升級為正式 SLA 時進行合約修訂。
