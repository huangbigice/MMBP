# 回測系統增強功能摘要

## 更新日期
2026-02-12

---

## 功能概覽

本次更新為 MMBP 回測系統新增了以下重要功能：

1. **統一參數配置系統（YAML）**
2. **調高目標波動率至 15-20%**
3. **放寬 Regime 門檻（ADX 降至 15）**
4. **動量濾鏡功能**
5. **增強壓力測試（多時期場景）**
6. **擴展 Walk-Forward 驗證（多資產支援）**
7. **新增 API 端點**

---

## 詳細變更

### 1. 統一參數配置系統

**目的**：將所有硬編碼參數提取到 YAML 配置檔，便於版本控制與參數調整。

**變更內容**：

- **新增檔案**：
  - `config/strategy_config.yaml`：策略參數配置檔
  - `config/__init__.py`：配置模組導出

- **擴展檔案**：
  - `config/config_loader.py`：新增 `StrategyConfig` dataclass 與 `load_strategy_config()` 方法

- **配置內容**：
  ```yaml
  strategy:           # 策略核心參數
  regime:             # Regime 分類參數
  volatility_targeting:  # 波動率目標參數
  performance_targets:   # 預期報酬與風險目標
  backtest:           # 回測通用參數
  momentum_filter:    # 動量濾鏡參數
  portfolio:          # 組合風險管理參數
  walk_forward:       # Walk-Forward 驗證參數
  monte_carlo:        # Monte Carlo 壓力測試參數
  logging:            # 日誌與除錯參數
  ```

### 2. 調整核心策略參數

**目的**：提升資本效率，追求 8-12% 年化報酬目標。

**變更內容**：

- **目標波動率**：從 10% 提升至 18%（預設值）
  - 檔案：`config/strategy_config.yaml`、`services/risk/position_sizing.py`
  
- **ADX 門檻**：從 20.0 降至 15.0（放寬趨勢判定）
  - 檔案：`config/strategy_config.yaml`、`services/backtest/regime.py`

- **替換硬編碼常數**：
  - 檔案：`services/backtest_service.py`
  - 所有硬編碼參數（`DEFAULT_THRESHOLD`、`BACKTEST_FUND_SCORE`、`ATR_STOP_MULTIPLIER` 等）改為從配置讀取

**預期效果**：

- 策略年化波動率接近 15-20%
- 進場次數增加 20-30%（ADX 門檻降低）
- 年化報酬目標 8-12%（視市場環境）
- Sharpe Ratio 維持或提升

### 3. 新增動量濾鏡功能

**目的**：過濾假突破，提升多單進場品質。

**變更內容**：

- **檔案修改**：`services/backtest_service.py`
  - `run_backtest()` 新增 `momentum_filter` 參數
  - `_build_backtest_df()` 實作動量濾鏡邏輯

- **濾鏡邏輯**：
  ```python
  if momentum_filter:
      momentum_ok = df["return_5"] > threshold  # 預設 threshold = 0.0
      df["signal_long"] = (df["signal_long"] & momentum_ok).astype(int)
  ```

- **注意事項**：
  - 只過濾多單，空單邏輯不變
  - 可透過 API 參數動態啟用/停用

**預期效果**：

- Walk-Forward 測試中，負 Sharpe 窗口比例 < 40%（原策略 50%）
- Sharpe 標準差降低 10-15%（提升穩定性）
- 最大連續虧損期縮短

### 4. 增強壓力測試功能

**目的**：測試策略在歷史危機時期（如 2022 熊市、2020 COVID 崩盤）的表現。

**變更內容**：

- **檔案修改**：`services/backtest/stress.py`

- **新增場景定義**：
  ```python
  CRISIS_SCENARIOS = {
      "2022_bear":        # 2022 年熊市
      "2020_covid":       # 2020 COVID 崩盤
      "2018_correction":  # 2018 年修正
      "2008_crisis":      # 2008 年金融海嘯
  }
  ```

- **新增函數**：
  - `stress_test_period()`：對特定歷史時期執行壓力測試
  - `stress_test_multiple_periods()`：批次測試多個危機時期
  - `analyze_stress_distribution()`：分析 Monte Carlo 壓力測試結果分佈

- **回傳指標**：
  - 總報酬、最大回撤、波動率、Sharpe Ratio
  - MDD 分位數、VaR、CVaR
  - 極端回撤機率

**使用範例**：

```python
from services.backtest import stress_test_multiple_periods

stress_results = stress_test_multiple_periods(
    df=backtest_df,
    scenarios=["2022_bear", "2020_covid"],
)
print(stress_results)
```

**壓力測試通過標準**：

- 2022 年熊市：MDD < -10%（極佳），-10% ~ -15%（合格）
- 2020 COVID：MDD < -20%
- 2018 修正：MDD < -12%
- Monte Carlo：至少 70% 模擬路徑 MDD > -10%

### 5. 擴展 Walk-Forward 驗證功能

**目的**：支援多資產組合的 Walk-Forward 驗證與策略比較。

**變更內容**：

- **檔案修改**：`services/backtest/walk_forward.py`

- **新增函數**：
  - `run_walk_forward_portfolio()`：多資產組合的 Walk-Forward 驗證
  - `compare_walk_forward_strategies()`：比較不同策略變體的表現

- **策略比較**：
  1. 基準策略（無動量濾鏡）
  2. 動量濾鏡策略（return_5 > 0）
  3. 多資產策略（如加入 0050.TW）
  4. 動量濾鏡 + 多資產策略

- **輸出指標**：
  - Sharpe 平均值與標準差
  - 負 Sharpe 窗口比例
  - 平均年化報酬與 MDD
  - 策略比較摘要表

**使用範例**：

```python
from services.backtest import compare_walk_forward_strategies

comparison = compare_walk_forward_strategies(
    symbol="2330.TW",
    start="2010-01-01",
    end="2025-12-31",
    backtest_service=backtest_service,
    portfolio_symbols=["0050.TW"],
)

print(comparison["summary"])
```

**預期效果**：

- 多資產組合波動率降低 15-20%
- MDD 縮小至少 20%
- Sharpe Ratio 提升 15-25%

### 6. API 層更新

**目的**：提供新功能的 HTTP API 端點。

**變更內容**：

- **檔案修改**：`api/routes.py`、`api/schemas.py`

- **更新端點**：
  - `GET /api/v1/backtest`：新增 `momentum_filter` 參數，`target_vol_annual` 改為可選（預設從配置讀取）

- **新增端點**：
  - `POST /api/v1/backtest/stress-test`：執行多時期壓力測試
  - `POST /api/v1/backtest/walk-forward-compare`：比較不同策略變體的 Walk-Forward 表現

- **新增 Schema**：
  - `StressTestRequest`、`StressTestResponse`
  - `WalkForwardCompareRequest`、`WalkForwardCompareResponse`

**API 使用範例**：

```python
import requests

# 壓力測試
response = requests.post(
    "http://localhost:8000/api/v1/backtest/stress-test",
    json={
        "symbol": "2330.TW",
        "scenarios": ["2022_bear", "2020_covid"],
    }
)

# Walk-Forward 比較
response = requests.post(
    "http://localhost:8000/api/v1/backtest/walk-forward-compare",
    json={
        "symbol": "2330.TW",
        "start": "2010-01-01",
        "end": "2025-12-31",
        "portfolio_symbols": ["0050.TW"],
    }
)
```

### 7. 主應用程式更新

**變更內容**：

- **檔案修改**：`main.py`

## 使用指南

### 1. 修改策略參數

編輯 `config/strategy_config.yaml`，調整關鍵參數後重啟應用程式。

詳見：`docs/策略參數調整指南.md`

### 2. 啟用動量濾鏡

**方式 A：API 參數**

```python
result = backtest_service.run_backtest(
    symbol="2330.TW",
    momentum_filter=True,
)
```

**方式 B：配置檔預設**

```yaml
momentum_filter:
  enabled: true
```

### 3. 執行壓力測試

```python
from services.backtest import stress_test_multiple_periods

df = backtest_service.get_backtest_df(symbol="2330.TW")
stress_results = stress_test_multiple_periods(df)
```

## 注意事項

1. **模型檔案**：驗證腳本需要已訓練的模型檔案（`rf_model_2330.pkl`）。
2. **網路連線**：需連線至 Yahoo Finance 下載股票資料。
3. **執行時間**：Walk-Forward 驗證可能需要數分鐘。
4. **配置修改**：修改 YAML 配置後需重啟應用程式。
5. **參數合理性**：避免極端參數值（如 target_vol > 50%）。
