# 策略參數調整指南

## 概述

本指南說明 MMBP 回測系統的參數配置與調整方式。所有策略參數現已統一由 YAML 配置檔管理，便於版本控制與參數優化。

---

## 配置檔案位置

**主配置檔**：`config/strategy_config.yaml`

此檔案包含所有策略核心參數，包括：
- 策略參數（模型門檻、ATR 停損等）
- Regime 分類參數（ADX 門檻、MA 週期等）
- 波動率目標參數
- 預期報酬與風險目標
- 回測通用參數
- 動量濾鏡參數
- 組合風險管理參數
- Walk-Forward 驗證參數
- Monte Carlo 壓力測試參數

---

## 關鍵參數說明

### 1. 波動率目標參數

```yaml
volatility_targeting:
  default_target_annual: 0.18    # 預設目標年化波動率（18%）
  vol_lookback: 20               # 波動率估計回望期
  max_leverage: 1.0              # 最大槓桿
  min_vol_floor: 0.01            # 波動率下限
```

**調整建議**：
- `default_target_annual`：提升至 15-20% 可提高資本效率，但需注意風險
- 較高的目標波動率適合追求 8-12% 年化報酬的策略
- 建議透過 Walk-Forward 驗證調整後的效果

### 2. Regime 分類參數

```yaml
regime:
  adx_threshold: 15.0            # ADX 趨勢強度門檻（從 20 降至 15）
  adx_period: 14                 # ADX 計算週期
  ma_period: 200                 # 均線週期
  slope_window: 5                # MA 斜率計算窗口
```

**調整建議**：
- `adx_threshold` 降低可增加進場機會，但可能增加假訊號
- 15-20 之間的值適合大多數市場環境
- ADX < 15 視為盤整，ADX 15-25 為開始出現趨勢，ADX > 25 為強趨勢

### 3. 預期報酬與風險目標

```yaml
performance_targets:
  expected_return_min: 0.08      # 預期年化報酬下限（8%）
  expected_return_max: 0.12      # 預期年化報酬上限（12%）
  max_drawdown_target: 0.10      # 最大回撤目標（10%）
```

**說明**：
- 這些參數用於評估策略是否達標
- 實際報酬受市場環境影響，需定期檢視調整
- MDD < 10% 為極佳表現，10-15% 為合格，> 15% 需優化

### 4. 動量濾鏡參數

```yaml
momentum_filter:
  enabled: false                 # 是否預設啟用動量濾鏡
  lookback_days: 5               # 動量回望期（return_5）
  threshold: 0.0                 # 動量門檻（>0 表示上漲）
```

**使用方式**：
- 動量濾鏡要求過去 N 日為正報酬才允許多單進場
- 可透過 API 參數動態啟用/停用
- 適合過濾假突破，提升勝率

## 使用方式

### 1. 修改配置檔

直接編輯 `config/strategy_config.yaml`：

```yaml
# 範例：提高目標波動率並放寬 ADX 門檻
volatility_targeting:
  default_target_annual: 0.20    # 提升至 20%

regime:
  adx_threshold: 12.0            # 降至 12（更激進）
```

修改後重啟應用程式即可生效。

### 2. 透過 API 動態調整

部分參數可透過 API 呼叫時覆寫：

```python
import requests

# 使用自訂波動率目標
response = requests.get(
    "http://localhost:8000/api/v1/backtest",
    params={
        "symbol": "2330.TW",
        "target_vol_annual": 0.20,  # 覆寫預設值
        "momentum_filter": True,     # 啟用動量濾鏡
    }
)
```

### 3. 在程式碼中使用

```python
from config import ConfigLoader

# 載入配置
config_loader = ConfigLoader()
strategy_config = config_loader.load_strategy_config()

# 取得參數
target_vol = strategy_config.volatility_targeting.default_target_annual
adx_threshold = strategy_config.regime.adx_threshold

print(f"目標波動率: {target_vol:.1%}")
print(f"ADX 門檻: {adx_threshold}")
```

## 參數優化流程

### 步驟 1：設定目標

明確策略目標（如年化報酬 10%、Sharpe > 1.5、MDD < 10%）。

### 步驟 2：調整參數

從配置檔修改關鍵參數：
- `target_vol_annual`：影響倉位大小與資本效率
- `adx_threshold`：影響進場頻率與訊號品質
- `momentum_filter.enabled`：影響多單進場條件

### 步驟 3：Walk-Forward 驗證

使用 Walk-Forward 驗證參數調整效果：

```python
from services.backtest import compare_walk_forward_strategies

comparison = compare_walk_forward_strategies(
    symbol="2330.TW",
    start="2010-01-01",
    end="2025-12-31",
    backtest_service=backtest_service,
    train_years=5,
    test_years=1,
    portfolio_symbols=["0050.TW"],
)

print(comparison["summary"])
```

### 步驟 4：壓力測試

對調整後的策略執行壓力測試：

```python
from services.backtest import stress_test_multiple_periods

df = backtest_service.get_backtest_df(
    symbol="2330.TW",
    start="2018-01-01",
    end="2024-12-31",
)

stress_results = stress_test_multiple_periods(
    df=df,
    scenarios=["2022_bear", "2020_covid"],
)

print(stress_results)
```

### 步驟 5：評估與迭代

根據驗證結果評估：
- Sharpe Ratio 是否提升
- 負 Sharpe 窗口比例是否降低
- MDD 是否符合目標
- 策略穩定性（Sharpe 標準差）是否改善

根據評估結果調整參數並重複步驟 2-4。

## 推薦參數組合

### 保守型（低波動，穩健報酬）

```yaml
volatility_targeting:
  default_target_annual: 0.12

regime:
  adx_threshold: 20.0

momentum_filter:
  enabled: true
```

**特點**：較低波動率、較高 ADX 門檻、啟用動量濾鏡
**預期**：年化報酬 6-8%，MDD < 8%，Sharpe > 1.2

### 積極型（高波動，追求高報酬）

```yaml
volatility_targeting:
  default_target_annual: 0.20

regime:
  adx_threshold: 12.0

momentum_filter:
  enabled: false
```

**特點**：較高波動率、較低 ADX 門檻、不使用動量濾鏡
**預期**：年化報酬 10-15%，MDD 可能達 12-15%，Sharpe > 1.0

### 平衡型（中等波動，風險調整後報酬最佳）

```yaml
volatility_targeting:
  default_target_annual: 0.18

regime:
  adx_threshold: 15.0

momentum_filter:
  enabled: true
```

**特點**：中等波動率、適中 ADX 門檻、動量濾鏡提升勝率
**預期**：年化報酬 8-12%，MDD < 10%，Sharpe > 1.5

## 常見問題

### Q1: 為什麼調高 target_vol_annual 後報酬沒有等比例提升？

A: 波動率目標影響的是倉位大小，但報酬同時受市場環境、訊號品質等因素影響。較高的波動率目標可能在趨勢市場表現更好，但在盤整市場可能增加虧損。

### Q2: ADX 門檻設多少最好？

A: 沒有固定答案，需根據市場特性與策略風格調整：
- 趨勢跟隨策略：20-25（避免假突破）
- 快速反應策略：12-18（增加進場機會）
- 保守穩健策略：25-30（只在明確趨勢時進場）

### Q3: 動量濾鏡一定有用嗎？

A: 不一定。動量濾鏡可提升勝率但可能降低交易頻率。建議透過 Walk-Forward 驗證在特定市場與時期的效果。

### Q4: 如何確保參數調整不是過度配適（overfitting）？

A: 使用以下方法驗證：
1. Walk-Forward 驗證（滾動訓練/測試）
2. 多時期壓力測試（測試不同市場環境）
3. Monte Carlo 模擬（測試報酬分佈穩定性）
4. 參數合理性檢查（避免極端參數值）

### Q5: 參數調整後需要重新訓練模型嗎？

A: 不需要。這些參數主要影響倉位管理與進場條件，模型預測邏輯不變。如需調整模型本身（特徵、演算法等），才需要重新訓練。
