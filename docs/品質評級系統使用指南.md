## 系統概覽

### 設計理念

品質評級系統採用「**查詢自由 + 智慧篩選**」雙軌制：

- ✅ **保留功能**：用戶仍可搜尋任何台股並查看完整分析
- ✨ **新增功能**：系統自動評級（A-F），組合策略只採用優質股
- 🎯 **目標提升**：組合 Sharpe Ratio 從 0.92 提升至 1.2+

### 核心特色

1. **基於實證數據**：評級根據最近 3 年回測 Sharpe Ratio 計算
2. **五級評分系統**：A（核心持倉）到 F（避免交易）
3. **智慧替代建議**：低評級股票自動推薦優質替代
4. **組合自動篩選**：投資組合只納入 A/B 級優質股

## 評級標準

### 評級定義

| 評級 | Sharpe 門檻 | 標籤 | 顏色 | 組合資格 | 說明 |
|------|-------------|------|------|----------|------|
| **A** | ≥ 1.0 | 核心持倉 | 🟢 綠色 | ✅ 可納入 | 優異的風險調整報酬，適合作為組合核心 |
| **B** | ≥ 0.5 | 穩定配置 | 🔵 藍色 | ✅ 可納入 | 穩定的投資標的，適合組合配置 |
| **C** | ≥ 0.0 | 觀察標的 | 🟡 黃色 | ❌ 不建議 | 表現平平，建議觀察或小額配置 |
| **D** | ≥ -0.5 | 謹慎操作 | 🟠 橙色 | ❌ 不建議 | 歷史表現欠佳，需謹慎評估風險 |
| **F** | < -0.5 | 避免交易 | 🔴 紅色 | ❌ 不建議 | 風險調整報酬極差，強烈不建議交易 |

### 計算方法

評級基於以下參數計算：

```python
# 回測期間：最近 3 年
# Sharpe 計算公式：(年化報酬 - 無風險利率) / 年化波動率
# 無風險利率：2%（與系統配置一致）
# 年化基準：252 交易日
```

## API 使用

### 1. 取得股票評級

**端點**：`GET /api/v1/stock/{symbol}/rating`

**參數**：
- `symbol`（必填）：股票代碼，如 `2330.TW`
- `start`（可選）：回測開始日期 `YYYY-MM-DD`，預設為最近 3 年
- `end`（可選）：回測結束日期 `YYYY-MM-DD`，預設為今天

**範例請求**：

```bash
# 基本查詢
curl http://localhost:8000/api/v1/stock/2330.TW/rating

# 指定時間範圍
curl "http://localhost:8000/api/v1/stock/3481.TW/rating?start=2021-01-01&end=2023-12-31"
```

**回應範例（A 級股票 - 台積電）**：

```json

  "symbol": "2330.TW",
  "rating": "A",

  "warning": false,
  "description": "優異的風險調整報酬，適合作為組合核心",
  "alternatives": null
```

**回應範例（F 級股票 - 群創）**：

## 前端整合

### 1. TypeScript 類型定義

```typescript
// src/api/types.ts
export interface StockRatingResponse {
  symbol: string;
  rating: 'A' | 'B' | 'C' | 'D' | 'F';
  label: string;
  color: string;
  sharpe_ratio: number | null;
  portfolio_eligible: boolean;
  warning: boolean;
  description: string;
  alternatives?: AlternativeStock[];
}

export interface AlternativeStock {
  symbol: string;
  name: string;
  category: string;
  rating: string;
  sharpe: number | null;
}
```

### 2. API 調用

```typescript
// src/api/stockApi.ts
import { fetchStockRating } from './stockApi';

// 取得評級
const rating = await fetchStockRating('2330');
console.log(rating.rating); // "A"
console.log(rating.label);  // "核心持倉"
```

### 3. React 組件使用

```tsx
// 在 StockAnalysis 組件中
const [rating, setRating] = useState<StockRatingResponse | null>(null);

useEffect(() => {
  if (!stockCode) return;
  
  fetchStockRating(stockCode)
    .then(setRating)
    .catch(console.error);
}, [stockCode]);

// 顯示品質標籤
{rating && (
  <div className={`quality-badge ${rating.color}`}>
    <span>{rating.rating}級 [{rating.label}]</span>
    {rating.sharpe_ratio !== null && (
      <span>Sharpe: {rating.sharpe_ratio.toFixed(2)}</span>
    )}
  </div>
)}

// 顯示警告與替代建議
{rating?.warning && (
  <div className="warning-banner">
    <p>⚠️ 此標的風險較高，不建議納入投資組合</p>
    {rating.alternatives?.map(alt => (
      <button key={alt.symbol} onClick={() => switchStock(alt.symbol)}>
        {alt.symbol} ({alt.rating}級)
      </button> 
  </div>
)}
```

## 智慧組合建構

### Python API 使用

```python
from services.rating_service import RatingService
from services.risk.portfolio import build_smart_portfolio

# 初始化評級服務
rating_service = RatingService(backtest_service=backtest_service)

# 候選股票（包含優質股與劣質股）

    "2330.TW",  # 台積電（A 級）
    "2454.TW",  # 聯發科（A 級）
    "2317.TW",  # 鴻海（B 級）
    "3481.TW",  # 群創（F 級，會被排除）
    "0050.TW",  # 元大台灣50（B 級）
]

# 建構智慧組合（只選 A/B 級，取前 3 檔）

    candidate_symbols=candidates,
    rating_service=rating_service,

)

# 結果範例

    print(f"{stock['symbol']}: {stock['rating']}級, "

          f"權重 {stock['suggested_weight']:.1%}")

# 2330.TW: A級, Sharpe 1.25, 權重 40.0%
# 2454.TW: A級, Sharpe 1.10, 權重 35.0%
# 2317.TW: B級, Sharpe 0.65, 權重 25.0%
# （3481.TW 被自動排除）
```

## 常見問題

### Q1: 評級多久更新一次？

**A**: 目前為**實時計算**（每次查詢時計算）。後期可優化為每日批次更新並快取結果。

### Q2: 評級計算需要多久？

**A**: 單一股票評級約需 **5-15 秒**（包含回測計算）。批次評級會依股票數量增加時間。

### Q3: 如果股票資料不足怎麼辦？

**A**: 資料不足時會返回 **F 級**，並標註「評級計算失敗」。建議增加回測期間或檢查資料來源。

### Q4: 可以自訂評級標準嗎？

**A**: 可以！編輯 `config/stock_quality.yaml`，調整各等級的 `min_sharpe` 門檻值。

範例：

```yaml
quality_tiers:
  A:
    min_sharpe: 1.2  # 提高 A 級門檻
```

### Q5: 如何新增優質股票池？

**A**: 編輯 `config/stock_quality.yaml` 的 `default_quality_stocks` 區塊：

```yaml
default_quality_stocks:
  - symbol: "2412.TW"
    name: "中華電"
    category: "電信"
    expected_rating: "B"
```

### Q6: 評級結果不符預期怎麼辦？

**A**: 可能原因與解決方案：

1. **回測期間太短**：增加 `backtest_years` 參數（預設 3 年）
2. **市場環境變化**：調整回測時間範圍（start/end 參數）
3. **波動率目標影響**：檢查 `use_vol_targeting` 設定
4. **資料品質問題**：驗證 yfinance 資料完整性
