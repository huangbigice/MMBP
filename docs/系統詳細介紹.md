## 1. 系統概觀

MMBP 是**股票投資決策支援後端**，主要功能包括：

- **資料**：透過 yfinance 取得歷史股價，並計算技術指標與大盤市場狀態特徵。
- **模型**：使用 **RandomForest 三類分類**（強勢 / 弱勢 / 趨勢不明），輸出 P(強勢) 作為連續倉位依據。
- **回測**：單資產與多資產組合回測，支援波動率目標倉位與逆波動率權重。
- **API**：FastAPI 提供預測、歷史資料、回測、LLM 對話等介面，供前端（如 FMFA）呼叫。

**重要對齊原則**：回測與線上預測使用的**特徵順序、Scaler、部位公式**必須與訓練腳本 `train_model第六版.py` 完全一致，否則績效會失真。

## 3. 模型訓練方式

### 3.1 標籤建構（labeling.py）

- **build_quality_labels**（訓練第六版採用）：  
  依 **未來 N 日報酬**（如 `future_return_30`）在**同一交易日橫斷面**排名：  
  - 前 `top_pct`（如 18%）→ 1（強勢/長期持有）  
  - 後 `bottom_pct`（如 18%）→ 0（弱勢/不建議）  
  - 其餘 → 2（趨勢不明/觀望）  
  當日樣本數少於 `min_stocks_per_date` 的日期整日剔除。

- **build_absolute_labels**：依絕對報酬門檻（如 >10% → 1，<-5% → 0）標註，供對照。

### 3.2 特徵與大盤 Regime（train_model第六版 + market_regime）

- **個股特徵**（與 `FEATURES` 一致）：  
  `return_1`, `return_5`, `return_20`, `rsi_14`, `rsi_120`,  
  `ma5_close`, `ma20_close`, `ma60_close`,  
  `volume_spike`, `price_strength`, `volatility_regime`,  
  以及大盤 **MKT_REGIME_FEATURES**（見下方技術指標一節）。

- **資料準備**：  
  - 從 `stock_list.json` 讀取股票列表，yfinance 下載。  
  - 價格 1%–99% 分位裁剪（與 DataService 一致）。  
  - 計算 MA、return、RSI、ma*_close、price_strength、volatility_regime、volume_spike、future_return_30。  
  - **build_quality_labels** 建標籤。  
  - 下載大盤 ^TWII，**compute_market_regime_features** → **merge_regime_into_panel**，再 dropna(MKT_REGIME_FEATURES)。

### 3.3 訓練流程（train_and_evaluate）

- 時間切分：`date < TRAIN_END` 為訓練集，`date >= TEST_START` 為測試集（TRAIN_END/TEST_START 在腳本內設定）。  
- **StandardScaler**：fit 僅 on 訓練集，transform 訓練與測試。  
- **RandomForestClassifier**：n_estimators=800, max_depth=15, min_samples_split=100, class_weight={0:2.5, 1:2.5, 2:1}。  
- 存檔：`joblib.dump({"model": model, "scaler": scaler, "features": FEATURES}, MODEL_PATH)`。  
- 評估：classification_report、測試集上 proba_strong / 回測用欄位。

### 3.4 Walk-Forward 滾動重訓（walk_forward_train.py）

- **generate_training_windows**：依 `data_start`、`data_end`、`train_years`、`roll_months` 產生滾動訓練視窗。  
- **train_one_window**：在單一視窗內切訓練集、fit RF、寫入指定 model_path。  
- **run_latest_window**：用最近 train_years 年資料訓練一次並覆寫模型檔，適合排程（如每 3 個月）執行。

## 4. 回測邏輯與公式

以下與 `docs/回測規則與公式說明.md` 及 `BacktestService._build_backtest_df` 對齊。

### 4.1 資料與特徵

- 價格：yfinance，台股 `.TW`；**1%–99% 分位裁剪**（與訓練一致）。  
- 特徵：由 **DataService.add_indicators** + **add_market_regime** 產生，特徵列表以 **ModelLoader.get_features()** 為準（即訓練時存下的 FEATURES），順序不可變。  
- 缺失值：送入模型前 `fillna(0)`；若有 **scaler**，用 **ModelLoader.get_scaler()** 做 transform。

### 4.2 模型訊號

- **proba_strong** = 類別 1（強勢）的機率 = `model.predict_proba(X_scaled)[:, idx_strong]`。  
- **TRAIN_PROBA_THRESHOLD = 0.55**：僅用於「進場次數」等統計（signal_long = proba_strong > 0.55），**不直接驅動部位**。  
- **Null 測試**：`null_mode="shuffled"` 打亂 proba_strong；`null_mode="hold"` 強制 proba_strong=0。

### 4.3 單資產部位與報酬（對齊訓練模型）

- **MAX_POSITION = 0.5**，**LOSS_THRESHOLD = -0.05**。  
- **前一日機率決定今日部位**（避免前視）：  
  `position_t = clip(proba_strong_{t-1}, 0, MAX_POSITION)`，首日填 0。  
- **未啟用波動率目標時**：`weight_t = position_t`。  
- **策略報酬**：  
  - 原始：`raw_t = weight_{t-1} * return_1_t`。  
  - 單日停損：若 `return_1_t < -0.05`，當日策略報酬 = 0。  
  - 手續費：`turnover_t = |weight_t - weight_{t-1}|`，`strategy_return_t = raw_t - turnover_t * fee_rate`（fee_rate 來自 config，如 0.0015）。  
- **累積與回撤**：`cum_return = (1 + strategy_return).cumprod()`，`drawdown = cum_return / cum_return.cummax() - 1`。

### 4.4 波動率目標倉位（可選）

- **VolTargetConfig**：target_vol_annual、vol_lookback、max_leverage、min_vol_floor。  
- 日波動率：`return_1.rolling(vol_lookback).std()`，年化 ×√252，並 clip 下限 min_vol_floor。  
- **compute_vol_target_weights**：raw_weight = target_vol_annual / vol_annual，再 clip 並乘以 position_binary。  
- **compute_confidence_vol_weights**：weight = vol_weight × confidence（做多時 confidence = proba_buy），本策略僅做多。  
- 要**完全對齊訓練模型**時，API 應傳 `use_vol_targeting=False`，此時 weight = position。

### 4.5 多資產組合回測

- 各標的先做單資產回測（可開波動率目標），得到每個標的的 **strategy_return** 日序列。  
- **align_returns**：日期聯集，缺日填 0。  
- **compute_inverse_vol_weights**：逆波動率權重，正規化且單一資產權重上限 max_single_weight，得到每日權重。  
- **aggregate_portfolio_returns**：組合報酬 = 前一日權重 × 當日各資產報酬；可選組合層級波動率目標縮放（用前一日 scale）。

### 4.6 績效指標（_result_from_df）

- 年化報酬：`total_return^(1/n_years) - 1`，n_years = len(df)/252。  
- 年化波動率：`std(strategy_return) * √252`。  
- 最大回撤：`min(drawdown)`。  
- 夏普：`(ann_return - risk_free_rate) / ann_vol`（ann_vol≤0 則 None）。  
- 交易次數：`signal_long.sum()`（proba_strong > 0.55 的天數）。

## 5. 虛擬下單與模擬交易

本系統**沒有獨立的「虛擬下單」或「紙上交易」模組**。  
**回測本身就是模擬交易**：在歷史資料上依模型訊號與規則計算每日倉位與報酬，不涉及真實委託與成交。

- **單資產回測**：`GET /api/v1/backtest` → BacktestService.run_backtest → _build_backtest_df 產生 strategy_return、cum_return、drawdown 等，即「若當時依此策略交易」的模擬績效。  
- **多資產組合回測**：`POST /api/v1/backtest/portfolio` → 各標的模擬報酬對齊後，以逆波動率權重聚合，同樣為模擬組合績效。  
- **壓力測試**（services/backtest/stress.py）：Monte Carlo、block bootstrap、單日衝擊、歷史危機情境，均為在歷史或模擬路徑上的**模擬**，不觸發真實下單。

若未來要實作「即時虛擬下單」（例如每日收盤後以當日訊號寫入虛擬持倉、隔日開盤虛擬成交），可在現有 BacktestService 與資料流基礎上，新增一個「虛擬帳戶」服務，依相同 position/weight 邏輯更新虛擬部位與損益；目前程式碼中未實作此層。

## 6. 技術指標

### 6.1 計算位置

- **DataService.add_indicators**（`services/data_service.py`）：用於 API 與回測的特徵計算，與訓練時定義一致。  
- **train_model第六版**：同一套指標用於訓練；RSI 由 **compute_rsi**（`train_model第六版`）計算，DataService 引用以保持一致。

### 6.2 指標一覽（與訓練/回測對齊）

| 欄位 | 公式 / 說明 |
|------|-------------|
| ma5, ma20, ma60, ma120, ma240 | close.rolling(w).mean() |
| return_1, return_5, return_20 | close.pct_change(1/5/20) |
| rsi_14, rsi_120, rsi_240, rsi_420 | compute_rsi(close, window)，RS=gain/loss，RSI=100-100/(1+RS)，NaN 填 50 |
| ma5_close, ma20_close, ma60_close | close / ma5、close/ma20、close/ma60 |
| price_strength | (close/close.shift(10)).rank(pct=True) |
| volatility_regime | return_1.rolling(20).std().rank(pct=True) |
| volume_spike | (volume / volume.rolling(20).mean()).clip(0, 3) |
| ema120, ema240, ema420, ema200 | close.ewm(span=s).mean() |

### 6.3 大盤市場狀態（MKT_REGIME_FEATURES）

大盤指數預設 **^TWII**，由 **market_regime.compute_market_regime_features** 計算後，**merge_regime_into_panel** 依 date 合併至個股：

| 欄位 | 公式 / 說明 |
|------|-------------|
| mkt_ma20_slope | 大盤 ma20.pct_change(5) |
| mkt_ma60_slope | 大盤 ma60.pct_change(5) |
| mkt_vol_60d | 大盤 close.pct_change().rolling(60).std() |
| mkt_drawdown_60 | (close - roll_max_60) / roll_max_60 |
| mkt_close_over_ma20 | (close/ma20) - 1 |
| mkt_close_over_ma60 | (close/ma60) - 1 |

回測與預測前必須成功 **add_market_regime**，否則 MKT_REGIME_FEATURES 缺失會導致模型輸出異常（proba_strong 過低等）。

## 8. 資料流與模組依賴關係

### 8.1 啟動流程（main.py lifespan）

```
ConfigLoader → AppConfig, load_strategy_config() → StrategyConfig
    ↓
get_model_version_info() → ModelVersionInfo
    ↓
ModelLoader(ModelLoadConfig(model_path)) → load() 可取得 model, features, scaler
    ↓
DataService() → fetch_stock_data, add_indicators, add_market_regime, required_features
    ↓
PredictionService(model_loader, data_service, model_version_info)
BacktestService(data_service, model_loader, model_version_info, strategy_config)
    ↓
app.state 掛載上述實例 + audit_logger
```

### 8.2 單資產回測呼叫鏈

```
GET /api/v1/backtest
  → BacktestService.run_backtest()
  → _build_backtest_df()
      → DataService.fetch_stock_data()
      → DataService.add_indicators()
      → DataService.add_market_regime()
      → ModelLoader.load() / get_features() / get_scaler()
      → 特徵 X = df[features].fillna(0)，順序 model_features
      → scaler.transform(X) → model.predict_proba() → proba_strong
      → position = proba_strong.shift(1).clip(0, 0.5)
      → weight = position 或 compute_confidence_vol_weights(...)
      → strategy_return（含單日停損與手續費）→ cum_return, drawdown
  → _result_from_df() → BacktestResult
```

### 8.3 預測呼叫鏈

```
POST /api/v1/predict
  → PredictionService 取得最新一筆資料
  → DataService.fetch_stock_data + add_indicators + add_market_regime
  → get_latest_fund_score（若仍使用）、required_features
  → ModelLoader.load()，X 依 model features 順序 + scaler
  → model.predict_proba() → proba_buy / 建議文字與評分
```

### 8.4 訓練與回測對齊要點

- 特徵順序：必須與 **train_model第六版.FEATURES** 及模型檔內儲存的 `features` 完全一致。  
- 標準化：推論與回測使用訓練時存下的 **scaler**，不可重新 fit。  
- 部位：`position = proba_strong.shift(1).clip(0, 0.5)`，weight 在未開波動率目標時等於 position。  
- 報酬：前一日 weight × 當日 return_1，單日 return_1 < -5% 當日報酬=0，再扣 turnover × fee_rate。  
- 資料前處理：價格 1%–99% 裁剪、RSI/大盤 regime 與訓練腳本一致。
