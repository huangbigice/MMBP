# POC（Proof of Concept）概念驗證文件

**文件編號**：POC-001  
**版本**：1.0  
**提案日期**：________  
**適用專案**：智慧股票投資決策支援系統（MMBP + FMFA）

---

## 一、文件目的

本 POC 文件旨在向潛在客戶或合作夥伴說明概念驗證之目的、範圍、成功準則與交付項目，供雙方於正式採購或合作前，就技術可行性與商業價值達成共識。

---

## 二、POC 概述

### 2.1 專案背景

智慧股票投資決策支援系統整合**機器學習模型**、**量化分析**與**技術指標視覺化**，為投資人提供結構化之決策輔助。本 POC 將驗證該系統於客戶指定情境下之適用性與效能表現。

### 2.2 驗證目標

| 目標 | 說明 |
|------|------|
| **技術可行性** | 確認 API 整合、資料格式、回應時間與穩定性能滿足客戶需求 |
| **功能完整性** | 驗證預測、回測、指標計算、LLM 輔助對話等核心功能符合預期 |
| **商業價值** | 透過實際使用情境評估系統對決策流程之助益 |

---

## 三、POC 範圍與邊界

### 3.1 納入範圍（In Scope）

- 後端 API 服務（MMBP）之佈署與連線驗證
- 前端分析介面（FMFA）之展示與操作
- 指定股票標的之歷史資料、技術指標、預測與回測功能
- LLM 投資助理對話（若客戶環境可連線 Ollama 或指定 LLM 端點）
- 基本效能與回應時間量測

### 3.2 排除範圍（Out of Scope）

- 客製化模型訓練或策略開發
- 生產環境高可用（HA）佈署與正式 SLA
- 大規模壓力測試或負載測試
- 法遵、資安稽核或正式合規認證

## 五、成功準則

POC 視為成功，須同時滿足以下條件：

| 編號 | 準則 | 驗證方式 |
|------|------|----------|
| SC-1 | API 健康檢查可正常回應 | 手動或自動呼叫 `GET /api/v1/health` |
| SC-2 | 至少 3 支指定標的可取得完整股價與指標資料 | 呼叫 `GET /api/v1/stock/{symbol}/indicators` |
| SC-3 | 預測 API 可回傳投資建議與評分 | 呼叫 `POST /api/v1/predict` |
| SC-4 | 單資產回測可產生權益曲線與績效指標 | 呼叫 `GET /api/v1/backtest` |
| SC-5 | 前端圖表可正確顯示資料並可切換標的 | 操作 FMFA 介面 |
| SC-6 | 主要 API 之 P95 回應時間 < 10 秒（POC 環境） | 簡單效能量測或日誌統計 |

## 六、時程與里程碑

| 階段 | 工作項目 | 預估天數 | 交付物 |
|------|----------|----------|--------|
| **Phase 1** | 環境準備、API 佈署、連線測試 | 3–5 天 | 可存取之 API 與前端 URL |
| **Phase 2** | 功能驗證、指定標的測試 | 3–5 天 | 功能驗證報告（含截圖） |
| **Phase 3** | 效能量測、客戶反饋收集 | 2–3 天 | POC 總結報告 |
| **總計** | — | **約 2 週** | 完整 POC 報告 |

> 實際時程得依客戶環境與資源配合度調整。

## 七、環境需求

### 7.1 供應方（我方）提供

- MMBP 後端服務佈署（開發或測試環境）
- FMFA 前端建置與佈署
- API 文件與基本操作說明
- 測試用 API Key 或存取權限（若已啟用認證）

### 7.2 客戶方配合

- 提供測試用股票標的清單
- 提供可存取 POC 環境之 IP 或網路條件（若有白名單需求）
- 指派對口窗口參與驗證與回饋
- 若有 LLM 需求，提供 Ollama 或相容 LLM 端點（或接受我方預設環境）

## 九、風險與假設

| 類型 | 內容 |
|------|------|
| **假設** | 客戶網路可連線至 POC 環境；金融資料來源（如 Yahoo Finance）於 POC 期間可正常取得資料 |
| **風險** | 外部資料源不穩定可能影響部分功能；LLM 服務若為遠端連線，延遲可能較高 |
| **緩解** | 事先確認資料源可用性；必要時以 mock 資料補充驗證 |
